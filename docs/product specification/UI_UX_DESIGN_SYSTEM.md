# FIND MY INTERIOR - UI / UX DESIGN SYSTEM

## Purpose

This document defines the design language of Find My Interior.

Every page must look like it belongs to the same product.

No page should feel like it was developed separately.

Every component should follow one visual system.

Consistency is more important than creativity.

---

# DESIGN PHILOSOPHY

Find My Interior is a professional marketplace.

It should feel

• Premium

• Modern

• Fast

• Trustworthy

• Clean

• Spacious

It should never feel

• Crowded

• Cheap

• Template Based

• Outdated

• Corporate ERP

---

# DESIGN PRIORITIES

Priority 1

Usability

Priority 2

Readability

Priority 3

Speed

Priority 4

Beauty

Beauty should never reduce usability.

---

# BUTTON RULES

Every button must perform an action.

No dead buttons.

Every button must show loading while waiting.

Every successful action should give confirmation.

Every failed action should explain why.

# ICONS

Use one icon library throughout.

Icons should always support text.

Never use icons alone.

---

# EMPTY STATES

Every empty state must answer

Why is this empty?

What should I do next?

Provide exactly one CTA.

Never show blank white screens.

---

# LOADING STATES

Every page should use

Skeleton Loaders

Loading Buttons

Progress Indicators

Never freeze the interface.

---

# SUCCESS STATES

After every successful action

Show

Toast

Status Update

UI Refresh

Never force manual refresh.

---

# ERROR STATES

Errors should explain

What happened

Why

How to fix it

Never display

500 Error

Undefined

Stack Trace

---

# NOTIFICATIONS

Use toast notifications for

Success

Warning

Error

Information

Never interrupt the workflow unnecessarily.

---

# MODALS

Use modals only when necessary.

Examples

Delete Confirmation

Award Project

Verification

Reject Reason

Never put large forms inside modals.

---

# DRAWERS

Use side drawers for

Quick Edit

Profile Preview

Messages

Never use fullscreen drawers on desktop.

# RESPONSIVENESS

Tablet

---

# ACCESSIBILITY

Every page should support

Keyboard Navigation

Screen Readers

Proper Contrast

Focus Indicators

Labels

---

# PAGE PERFORMANCE

Avoid unnecessary re-renders.

Lazy load heavy sections.

Paginate long lists.

Optimize images.

---

# CONSISTENCY RULES

Every page should use

Same Buttons

Same Cards

Same Forms

Same Tables

Same Modals

Same Empty States

Same Colors

Same Typography

If a component exists, reuse it.

Never redesign the same component twice.

---

# REMOVE

Remove

Dummy Content

Placeholder Text

Lorem Ipsum

Broken Images

Unused Buttons

Dead Links

Unused Sections

Static Fake Data

Everything visible should have a purpose.

---

# FINAL UX CHECK

Every page should answer

What is this page?

What can I do here?

What should I do next?

If the answer is not obvious within five seconds, redesign the page.

---

# SUCCESS CRITERIA

The platform should feel like one cohesive product.

A user should never feel they have navigated into another application.

Every interaction should be smooth, intuitive and purposeful.

Consistency should be visible across the entire platform.
