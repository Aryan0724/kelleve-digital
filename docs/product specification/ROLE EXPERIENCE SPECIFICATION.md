# ROLE EXPERIENCE SPECIFICATION

## Purpose

This document defines the complete experience of every user role on FindMyInterior.

This is the final behavioural specification.

From now on every page, dashboard, button, recommendation, search result and workflow must follow this document.

A user must only see features that are useful for that user.

Never expose functionality that belongs to another role.

The platform must feel like a dedicated product for every type of user instead of one generic marketplace.

---

# GENERAL RULES

Every role has its own:

* Homepage
* Dashboard
* Sidebar
* Search
* Recommendations
* Notifications
* Profile
* Verification
* Messages
* Opportunities

The UI must never show irrelevant widgets.

Never display buttons that the current user cannot use.

Never display empty sections.

Never display future features.

Everything visible must work.

---

# UNIVERSAL RULES

Every role should only see relevant opportunities.

Every homepage should be personalized.

Every dashboard should solve that user's daily work.

Every recommendation should match that role.

Every search should return only useful results.

Every CTA must lead to a working feature.

Never expose another role's workflow.

The user should always feel that the platform was built specifically for them.
